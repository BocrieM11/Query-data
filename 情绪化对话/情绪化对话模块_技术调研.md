## 二、核心论文与方法

### 2.1 情感支持对话框架（ESC — Emotional Support Conversation）

| 论文 | 出处 | 核心方法 | 关键指标 |
|------|------|---------|---------|
| **FPEMF** — Fine-Grained Problem Enhancement with Multi-Dimensional Feedback | EMNLP 2025 | 问题分类从13类扩展到45类细粒度；四维反馈评估（情感理解/策略有效性/上下文一致性/主题相关性）；提出EmoCare数据集（2.6K对话/42.8K句） | 在多个LLM主干上提升自动+人工评分 |
| **EmoFSM** — Finite State Machine for ESC | PAKDD 2026 / arXiv 2025.04 | FSM + LLM自推理：每轮自主分析求助者情绪→选择策略→生成回复，无需外部支持 | 超越Chain-of-Thought、微调、外部支持方法 |
| **COMPEER** — Controllable Empathetic Reinforcement Reasoning | 山东大学+快手, 2025 | GRPO强化学习 + UnifiReward统一奖励（同时覆盖过程奖励和结果奖励）；冗余感知奖励重加权惩罚重复回复；DISC行为模式人格感知改写 | 解决"熵坍塌"和回复重复问题 |
| **MultiAgentESC** | 2025 | 训练无关三阶段多代理协作：对话分析→策略商讨→回复生成；响应中心多代理投票解决策略选择一对多问题 | — |
| **IntentionESC** | ACL 2025 Findings | ICECoT意图中心思维链：情绪分析→意图推理→策略选择→生成回复；自动化标注流水线 | — |
| **ESC-Judge** | EMNLP 2025 | 基于Hill's E-I-A咨询模型（Exploration-Insight-Action）的自动评估框架；合成真实求助者角色+专用Judge LLM | 与博士级标注者85%/83%/86%一致率 |

### 2.2 共情回复生成（Empathetic Response Generation）

| 论文 | 出处 | 核心方法 |
|------|------|---------|
| **EmPO** — Emotion Grounding via Preference Optimization | EMNLP 2025 | 基于情感grounding构建理论驱动的偏好数据集，通过偏好优化对齐LLM；使用diff-EPITOME和BERTscore评估 |
| **APTNESS** — Appraisal Theory + Emotion Support Strategies | 2024.07 | 认知评价理论(Cognitive Appraisal Theory) + 情感支持策略 + 检索增强生成 |
| **CasDecNet** — Multi-Stage Cascading Information Fusion | DEXA 2025 | 三阶段解码网络：(1)上下文理解 → (2)认知&情感共情获取 → (3)策略支持融入；使用多维外部知识推理情境 |
| **Metamo** — Multi-Agent Cognitive Modeling | 2025 | 多代理认知建模；在EmpatheticDialogues和EDOS上超越GPT-3.5/GPT-4o-mini零样本表现 |

### 2.3 记忆与个性化机制

| 论文/专利 | 核心方法 | 亮点 |

## 三、关键技术方法总结

### 3.1 核心方法对比

| 方法 | 优点 | 适用场景 | 实现复杂度 |
|------|------|---------|-----------|
| **四阶段Prompt工程** | 无需训练，即插即用，效果提升显著(17-40%) | 快速落地验证 | **低** |
| **EmoDiag (Logit层情感控制)** | 不微调LLM，显存减半，即插即用的插件式方案 | 需要精细情感方向控制 | **中** |
| **LoRA微调** | 效果稳定，可控性强，参数高效 | 有标注数据的场景 | **中** |
| **GRPO强化学习 (COMPEER)** | 回复多样性好，避免重复，过程+结果双优化 | 追求高质量生成 | **高** |
| **Multi-Agent协作** | 专业化分工，灵活扩展，消除中心瓶颈 | 复杂多轮对话 | **高** |
| **EmoFSM状态机** | 结构化程度高，可解释性强 | 对话流程可控的场景 | **中** |

### 3.2 关键设计模式

#### 模式一：ICECoT 思维链推理

```
用户输入 → [情绪分析] → [意图推理] → [策略选择] → [回复生成]
              ↓              ↓            ↓            ↓
         情绪类别+强度   求助/闲聊/   共情回应/    最终自然语言
                        表达需求    提问引导/    回复
                                    提供信息
```

#### 模式二：Hill's E-I-A 咨询模型

```
Exploration（探索）  →  Insight（洞察）  →  Action（行动）
    ↓                      ↓                    ↓
理解老人的情绪和处境    帮助老人获得新认识    给出建设性建议或行动
"听起来您今天感到..."   "您有没有想过..."     "要不我们试试..."
```

#### 模式三：四阶段共情Prompt

```
阶段1: 建立信任关系 → "我在听，您慢慢说"
阶段2: 识别理解情绪 → "听起来您有些孤单"
阶段3: 苏格拉底式提问 → "您觉得什么能让您开心一点？"
阶段4: 摘要与行动建议 → "那我们一起听听您喜欢的戏曲吧"
```

## 五、技术架构建议

### 5.1 推荐的四层架构

```
┌──────────────────────────────────────────────────────────────┐
│                     Prompt策略层                              │
│  - System Prompt角色设定（"小乐"，温和年轻女性）                │
│  - 四阶段共情Prompt模板（信任→识别→提问→建议）                  │
│  - 老人画像动态注入（姓名/喜好/病史/家属/当日状态）              │
├──────────────────────────────────────────────────────────────┤
│                     记忆机制层                                │
│  - 短期记忆：Redis（最近N轮对话，滑动窗口）                     │
│  - 长期记忆：Milvus向量库（历史对话RAG检索，语义相似度匹配）      │
│  - 情感记忆：情感状态追踪向量（开心/低落/焦虑/孤独/平稳）         │
│  - 遗忘机制：Sentence-BERT语义相似度过滤冗余信息                │
├──────────────────────────────────────────────────────────────┤
│                     策略选择层                                │
│  - ICECoT思维链推理：情绪分析→意图推理→策略选择                 │
│  - 策略库：共情回应 | 苏格拉底提问 | 正向引导 | 提供信息 | 转移注意│
│  - 基于Hill's E-I-A模型：Exploration→Insight→Action            │
│  - EmoDiag Logit层情感方向控制（温暖/鼓励/镇定/活泼）            │
├──────────────────────────────────────────────────────────────┤
│                     生成与评估层                               │
│  - LLM主干：DeepSeek / Qwen-plus / GPT-4o                     │
│  - TTS合成：CosyVoice 2（支持情感语调+声音克隆）                │
│  - ASR识别：Paraformer（中文方言优化）                          │
│  - 质量评估：ESC-Judge自动评分 + 人工抽检                       │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 主动对话触发机制（借鉴绘话智能「小白」）

| 触发类型 | 触发条件 | 对话示例 |
|---------|---------|---------|
| **定时触发** | 早安(7:00)、午休提醒(13:00)、用药提醒(按计划)、晚安(21:00) | "王奶奶早上好！今天天气晴朗，15-22°C，适合去花园走走" |
| **事件触发** | IoT检测起床/离床/回房/未用餐 | "奶奶您起床啦，昨晚睡得好吗？记得先喝杯温水" |
| **情感触发** | 情感基线偏离（连续2天情绪低落） | "奶奶，这两天感觉您话少了些，有什么心事想聊吗？" |
| **记忆触发** | 特殊日期/历史对话引用 | "奶奶，上次您说孙子上周高考，成绩出来了吗？" |

### 5.3 情感状态追踪模型

```
情感维度（借鉴七维坐标系 + 老年人场景简化）：

维度1: 愉悦度 (valence)         ← →  开心 — 低落
维度2: 激活度 (arousal)        ← →  活跃 — 安静
维度3: 社交意愿 (sociability)  ← →  想聊天 — 想独处
维度4: 焦虑度 (anxiety)        ← →  平静 — 焦虑
维度5: 孤独感 (loneliness)     ← →  充实 — 孤独

每日更新基线 → 检测偏离 → 触发对应的对话策略调整
```

## 六、落地路线建议

### 第一阶段：基础共情对话（4-6周）

- [ ] 四阶段Prompt工程 + System Prompt角色设定
- [ ] Redis短期记忆 + Milvus长期记忆（RAG检索历史对话）
- [ ] 老人画像注入（姓名、喜好、病史、家属信息）
- [ ] 基础情感分析（NLP文本情感分类：正向/中性/负向）
- [ ] 定时主动问候（早安/晚安/用药提醒）

**关键技术**：Prompt工程 + LangChain Memory + RAG

### 第二阶段：深度情感能力（4-6周）

- [ ] 情感状态五维追踪模型
- [ ] ICECoT思维链推理（情绪分析→意图推理→策略选择）
- [ ] 四阶段共情对话（信任建立→情绪识别→苏格拉底提问→行动建议）
- [ ] 事件驱动的主动对话（IoT数据联动）
- [ ] EmoDiag Logit层情感方向控制

**关键技术**：情感计算 + 思维链推理 + IoT联动

### 第三阶段：个性化与多模态（4-6周）

- [ ] 语音情感识别（分析语速/音调/停顿判断情绪）
- [ ] CosyVoice 2情感化TTS（不同情绪不同语调）
- [ ] 家属声音克隆（用子女声音播报，增强亲切感）
- [ ] 遗忘机制（过滤冗余历史，保留关键记忆）
- [ ] ESC-Judge自动质量评估 + A/B测试

**关键技术**：语音情感识别 + 声音克隆 + 对话质量评估
