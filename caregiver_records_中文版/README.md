## 字段说明

### 基本标识

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 全局唯一标识符 | `REC-JA-0001` / `REC-AUG-0123` / `REC-REAL-0005` |
| `category` | string | 分类代码，2 位数字 | `01` ~ `09` |
| `category_label` | string | 分类名称（日文） | `日常ケア` / `異常_急変` |
| `language` | string | 语言代码 | `ja` |

### 数据来源标记

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | string | 数据生成来源描述 |
| `source_type` | string | `adapted_from_template`（合成）/ `real_record`（真实记录参考）/ `augmented_from_real`（基于真实结构扩增） |

### 机构与人员

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `facility_type` | string | 养老机构类型 | `特別養護老人ホーム「さくら苑」` |
| `resident_id` | string | 老人编号 | `R001` |
| `caregiver_name` | string | 护工姓名（虚构） | `田中花子` / `佐藤美咲` |

### 老人信息（resident_info）

同一 `resident_id` 出现在多条记录中，模拟同一老人的多日连续数据，可用于趋势分析。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `resident_id` | string | 老人编号 | `R001` |
| `age` | int | 年龄 | `88` |
| `gender` | string | 性别 | `男` / `女` |
| `care_level` | string | 护理等级 | `要介護1` ~ `要介護5` |
| `primary_condition` | string | 主要病症 | `Alzheimer型認知症` / `Parkinson病` |
| `room_number` | string | 房间号 | `201` |
| `mobility` | string | 移动能力 | `歩行器` / `車椅子` / `自立` / `全介助` |
| `communication` | string | 沟通能力 | `良好` / `中等度障害あり` / `やや困難` / `不可` |

### 时间与班次

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|

| `shift` | string | 当值班次 | `日中` / `夕方` / `夜勤` / `早番` |

### 记录正文

| 字段 | 类型 | 说明 |
|------|------|------|
| `record_text` | string | **核心字段（训练用）**。日文护工记录，口语化风格（模拟语音转文字）。每条唯一。 |
| `record_text_zh` | string | **中文翻译对照**。与 `record_text` 一一对应的中文翻译，用于人工审核。每条均有。 |

**日文示例（`record_text`）**：
```
【食事介助】朝食: お粥「全量」、味噌汁「8割」、焼き鮭「半分」。
自力でスプーン使用。むせなし。口腔ケア実施。
```

**对应中文翻译（`record_text_zh`）**：
```
【食事介护】早餐: 粥(全量)、味噌汤(8成)、烤鲑鱼(一半)。
自行使用勺子。无呛咳。口腔护理完成。
```

## 构建方法

### 第一步：模板合成（500 条）

从以下公开标准中提取 103 条记录模板，运行 `build_final.py` 随机填充生成：

| 标准 | 地区 | 提取内容 |
|------|------|----------|
| 厚生労働省介護記録標準フォーマット | 日本 | 口述式记录（食事·排泄·入浴·体位変換等） |
| DB36/T 1582-2022 | 中国·江西 | 护理记录表（附录C）、日常健康记录表（附录F） |
| DB50/T 1421-2023 | 中国·重庆 | 照护记录单（附录C）、压疮预防记录（附录G）、交接班记录（附录L） |
| 养老机构跌倒应急预案 | 中国 | 跌倒事件处置模板 |

每条记录分配虚构老人（24人池）、随机生成生命体征（在医学合理范围内）、确保 `record_text` 唯一。

### 第二步：真实记录收集（23 条）

从公开渠道收集真实/范例护理记录，经脱敏后整合：

| 来源 | 条数 |
|------|------|
| 上海 H 社会福利院个案（学术论文） | 2 |
| 养老护理日志/交接班/压疮公开模板范例 | 5 |
| 福山老年公寓跌倒事件记录 | 1 |
| DB36/T + DB50/T 国家标准附录 | 5 |
| 知乎/抖音/快手/B站 培训教材·护工口述 | 10 |

### 第三步：结构扩增（190 条）

基于 23 条真实记录的文本结构和专业风格，编写 14 类生成器（入院、跌倒、发热、压疮、交接班、康复、情绪、误吸、睡眠、工作日、外出、排尿、拒药、日常），生成 190 条新记录。全条经过医学数值范围校验和去重。

### 第四步：日语统一 + 中文翻译

- 删除英文 125 条
- 中文 463 条经文化适配转为日文（食物、活动、护理等级、称呼等全部日本本地化）
- 原始日文 125 条 + 转换后 463 条 = 588 条，全部附加 `record_text_zh` 中文翻译
- 另导出纯中文版 588 条到 `caregiver_records_中文版/`

### 质量保证

| 检查项 | 结果 |
|--------|------|
| 完全重复文本 | 0 条 |
| vital_signs 医学合理性 | 全部在正常范围内 |
| 居民档案一致性 | 同一 `resident_id` 属性无矛盾 |
| 文本与结构化字段对应 | 数值一致 |

## 使用场景

1. **LLM 语音转文字 + 结构化提取**：`record_text` → 口语化日文护工记录输入，`vital_signs` / `meal_intake` / `mood` 为提取标签
2. **Dashboard 数据分析**：同一 `resident_id` 跨日记录 → 血压趋势、体重变化、风险预警
3. **多语言验证**：`record_text_zh` 中文翻译用于人工审核和双语模型评估
